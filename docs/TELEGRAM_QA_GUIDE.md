## 🚨 신고 유형별 대응

### 텔레그램 신고 버튼 사용법

```
이미지 미리보기 → ⚠️ 신고 버튼 클릭 → 신고 유형 선택
```

### 신고 유형 3가지

| 신고 유형 | 설명 | 후속 조치 |
|----------|------|----------|
| **텍스트 중첩** | 텍스트가 햇살이 얼굴/음식 가림 | reoverlay (텍스트만 재작업) |
| **이미지 오류** | AI 품질 불량, 규칙 위반 | regenerate (이미지 재생성) |
| **기타** | 캡션 오류, 기타 문제 | 수동 확인 필요 |

## 📋 규칙 위반별 신고 방법

### 1. 본문 텍스트 규칙 위반

**증상:**
- 텍스트 위치가 규칙과 다름
- 폰트 크기/색상 오류
- 텍스트가 햇살이 얼굴 가림

**신고 방법:**
```
⚠️ 신고 → 텍스트 중첩 선택
```

**후속 조치:**
- reoverlay 스크립트 실행
- 텍스트만 재작업 (배경 이미지 유지)

---

### 2. CTA에 AI 이미지 사용

**증상:**
- 4번째 이미지(저장 & 공유)에 AI 생성 이미지 사용
- 규칙: CTA는 반드시 실사 햇살이 사진

**신고 방법:**
```
⚠️ 신고 → 이미지 오류 선택
```

**후속 조치:**
- CTA 이미지 재생성 또는 실사 교체
- regenerate 필요

---

### 3. 안전 색상 오류

**증상:**
- DANGER 음식인데 녹색 사용
- SAFE 음식인데 빨강 사용

**신고 방법:**
```
⚠️ 신고 → 이미지 오류 선택
```

**후속 조치:**
- 색상 수정 필요
- regenerate 또는 reoverlay

## 📋 규칙 참조

### RULES.md 텍스트 오버레이 규칙

```
1. 표지: 영문 대문자, 중앙 상단
2. 본문: 하단 1/3 영역, 햇살이 얼굴 가리지 않음
3. CTA: 실사 햇살이 사진 필수
4. 색상: 안전등급에 따른 색상 코드
```

### 안전 색상 코드

```
SAFE (안전):     #4CAF50 (녹색)
CAUTION (주의):  #FFD93D (노랑)
DANGER (위험):   #FF5252 (빨강)
```
