# 📘 텔레그램 콘텐츠 규칙 위반 대응 가이드 v1
**Project Sunshine · PD 운영 기준**

---

## 1️⃣ 이 가이드는 언제 쓰나?

텔레그램에서 아래와 같은 메시지가 나왔을 때:

- ❌ 본문 텍스트 규칙 위반
- ❌ CTA에 AI 이미지 사용
- ⚠️ 품질 경고 / 신고 필요

👉 **이 가이드는 "지금 승인해야 하나? 신고해야 하나?"를 고민하지 않게 만드는 게 목적**

---

## 2️⃣ 위반 유형별 즉시 판단표 (이거만 보면 됨)

| 위반 메시지 | 발생 빈도 | PD 액션 | 자동/수동 |
|------------|----------|---------|----------|
| 본문 텍스트 규칙 위반 | **약 95%** | ⚠️ 신고 → 텍스트 중첩 | 🤖 자동 |
| CTA에 AI 이미지 사용 | **약 5%** | ⚠️ 신고 → 이미지 오류 | 👤 수동 |
| 애매함 / 둘 다 의심 | 극소수 | ⚠️ 신고 → 기타 | 👤 판단 |

**👉 승인으로 커버하려 하지 말 것 → 신고가 기본값**

---

## 3️⃣ 텔레그램에서 실제로 누르는 버튼 가이드

### 🔴 Case A. 본문 텍스트 규칙 위반 (95%)

**판단 기준:**
- 글자가 많다
- 줄간격/위치가 규칙과 다르다
- 가독성 떨어짐
- **"어… 이거 좀 이상한데?" 느낌 들면 거의 이거임**

**PD 액션:**
```
⚠️ 신고
 → [텍스트 중첩]
```

**그 다음 자동으로 일어나는 일:**

1. 신고 기록 저장
2. RULES.md 기반 reoverlay 실행
3. 텍스트만 수정된 이미지 생성
4. 텔레그램으로: **"재작업 완료, 재검토 요청"**
5. PD는 다시 승인/재신고만 하면 됨

**👉 PD가 텍스트 수정 고민 ❌**
**👉 "잘 고쳐졌나?"만 보면 됨**

---

### 🔴 Case B. CTA에 AI 이미지 사용 (5%)

**판단 기준:**
- CTA 영역이:
  - 너무 AI 느낌
  - 실사여야 하는데 일러스트/합성 느낌
  - RULES.md에서 AI 금지된 영역

**PD 액션:**
```
⚠️ 신고
 → [이미지 오류]
```

**그 다음:**
- ❌ 자동 수정 안 됨
- ✅ 최부장에게 알림만 감
- 최부장이:
  - 실사 이미지 교체
  - CTA 다시 구성
- 완료 후 텔레그램으로: **"재작업 완료, 재검토 요청"**

**👉 이건 사람 손 타는 게 정상**
**👉 자동화 기대 ❌**

---

## 4️⃣ 절대 하면 안 되는 행동 TOP 3

### ❌ 1. "이번만 승인"
- 규칙 위반은 누적됨
- 나중에 브랜드 기준 망가짐

### ❌ 2. 실패 메시지 보고 재시도
- 이전 이슈처럼 **가짜 실패 → 중복 사고** 유발

### ❌ 3. 텔레그램 메시지 문구를 믿고 추정
- 항상 **이미지 기준**으로 판단
- 메시지는 참고용

---

## 5️⃣ PD 판단 원칙 (한 줄 요약)

```
조금이라도 규칙이 걸리면 승인하지 말고 신고한다
→ 자동화가 처리하게 둔다
→ PD는 최종 품질만 본다
```

---

## 6️⃣ 이 가이드의 설계 철학 (중요)

- **PD는 판단자, 작업자가 아님**
- 95%는 자동화가 해결해야 정상
- 5%만 사람이 개입
- **"신고 = 문제"가 아니라**
- **👉 신고 = 정상 프로세스**
