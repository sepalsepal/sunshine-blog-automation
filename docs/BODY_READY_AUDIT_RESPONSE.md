## 결론 먼저

- **게시 가능: 0건** → 정상
- 지금 승인하면 100% 사고
- 전부 신고 → 재작업 큐로 보내는 게 맞음

---

## 1️⃣ 심각도별 PD 액션 표 (이거 기준으로 누르면 됨)

### 🔴 심각 (즉시 재작업, 승인 금지)

| 콘텐츠 | 문제 유형 | PD가 누를 버튼 |
|--------|----------|---------------|
| potato | 한글 폰트 깨짐 (□□□) | ⚠️ 신고 → 이미지 오류 |
| kalguksu | CTA 텍스트 없음 | ⚠️ 신고 → 이미지 오류 |
| naengmyeon | CTA 텍스트 없음 | ⚠️ 신고 → 이미지 오류 |
| sandwich | CTA 텍스트 없음 | ⚠️ 신고 → 이미지 오류 |
| cheetos | CTA 없음 + 이미지 중복 | ⚠️ 신고 → 이미지 오류 |

**📌 이유 (데이터 기준)**
- 텍스트 오버레이 문제가 아님
- reoverlay로 절대 해결 안 됨
- 이미지 자체 재생성이 필요

**👉 전부 수동 처리 대상 (5% 케이스)**

---

### 🟡 경고 (자동 처리 가능)

| 콘텐츠 | 문제 유형 | PD 액션 |
|--------|----------|---------|
| onion | FORBIDDEN인데 노랑 사용 | ⚠️ 신고 → 텍스트 중첩 |
| garlic | 동일 이슈 예상 | ⚠️ 신고 → 텍스트 중첩 |

**📌 이유**
- 색상 규칙 위반은 → RULES.md 기반 reoverlay로 자동 수정 가능
- 실제 발생 빈도 95% 케이스

**👉 자동화 태우는 게 맞음**

---

## 2️⃣ PD가 오늘 해야 할 "정확한 클릭 순서"

### Step 1. 🔴 심각 5건 처리 (사람 손 필요)

텔레그램에서 각각:
```
⚠️ 신고
 → 이미지 오류
```

**📩 효과**
- 최부장에게 작업 알림
- CTA 실사 이미지 재제작
- 중복 이미지 제거
- 텍스트 포함 CTA 재구성

**❗ 여기서는 기다리는 게 맞음 → 오늘 게시 욕심 ❌**

---

### Step 2. 🟡 onion / garlic 자동 처리

```
⚠️ 신고
 → 텍스트 중첩
```

**📩 효과**
- reoverlay 자동 실행
- 색상: FORBIDDEN → 빨강
- 완료 후 재검토 요청 올라옴

**👉 이 2건만 오늘 게시 가능 후보**

---

## 3️⃣ "왜 게시 가능 0건이 정상인가" (중요)

데이터적으로 보면:

- 전체 8건 중
  - 🔴 5건: 이미지 구조 문제 (자동화 불가)
  - 🟡 2건: 규칙 위반 (자동화 가능)
  - ⚠️ 1건: 확인 필요

**👉 초기 body_ready 품질이 낮았던 것**
**👉 PD 판단 미스 아님**
**👉 전수조사한 지금이 오히려 정상화 시점**

---

## 4️⃣ 이 리포트로 확정해야 할 운영 원칙 (강제)

### ❌ 금지

- "CTA 텍스트 없는 상태로 승인"
- "나중에 고치고 일단 게시"
- "reoverlay로 다 해결될 거라 기대"

### ✅ 원칙

- CTA 문제 = 무조건 이미지 오류
- 색상/텍스트 규칙 = 텍스트 중첩
- body_ready라도 품질검수 통과 못 하면 ready 아님

---

## 5️⃣ 다음에 바로 써먹을 한 줄 룰

```
CTA가 비어 있으면 자동화 대상이 아니다
→ 무조건 이미지 오류 신고
```
