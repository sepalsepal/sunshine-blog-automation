### 완료 판정 조건

```
완료 판정 전 필수 확인:

4. 스타일 불일치 0건인가?

하나라도 NO → 완료 처리 금지
```

## 예외 처리 선언 규칙 (Day 12 추가)

### 원칙

기존 템플릿/규칙과 다른 방식으로 제작할 경우
**반드시** 아래 로그를 문서에 남긴다.

### EXCEPTION_LOG 양식

```markdown
## [EXCEPTION_LOG]

| 항목 | 내용 |
|------|------|
| 콘텐츠명 | {topic_en} |
| 예외 내용 | {기존 규칙과 다른 점 상세} |
| 판단자 | {담당자명} |
| 판단 시점 | {YYYY-MM-DD HH:MM} |
| 근거 | {왜 예외가 필요했는지} |
| PD 승인 | □ 사전 승인 / □ 사후 보고 |
```

### 예외 처리 규칙

```
1. EXCEPTION_LOG 없는 예외 처리 → 결과가 좋아도 FAIL
2. 예외는 반드시 사전 승인 또는 즉시 사후 보고
3. 예외 적용 콘텐츠는 별도 표시 (폴더명에 _EXCEPTION 추가)
4. 예외 사유가 반복되면 → 규칙 자체를 수정 (예외 제거)
```
