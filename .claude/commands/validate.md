# /validate - 수동 Validator 실행

## 인자
- `$ARGUMENTS` = 콘텐츠 번호 또는 이름 (선택)
- 인자 없으면 가장 최근 작업 콘텐츠 자동 감지

## 실행 절차

### 1. 대상 콘텐츠 특정
- `$ARGUMENTS` 있으면 해당 콘텐츠
- 없으면 `3_approved/` 또는 `2_body_ready/`에서 최근 수정 폴더

### 2. Validator 실행

#### 2.1 표지 검증 (§8 v2.0)
```
□ 해상도 1080x1080
□ 음식명 한글 텍스트 존재
□ 영어 텍스트 없음
□ 폰트: BlackHanSans-Regular 120px
□ 텍스트 색상: 흰색
□ 드롭 쉐도우: offset=2, alpha=100, blur=3
□ 상단 그라데이션: 25%, alpha 80→0
□ Y 위치: 80px
```

#### 2.2 C2 인포그래픽 검증 (§2.4 v2.0)
```
□ 해상도 1080x1080
□ 상단 그라데이션 헤더 존재
□ 헤더 제목 존재
□ 카드형 레이아웃 (항목별 박스)
□ 컬러풀 번호 아이콘
□ 항목별 제목 + 설명 구조
□ 하단 보조 요소 1개 이상
□ 텍스트 가독성 (배경 대비)
□ 폰트 크기 규칙 준수
□ 폰트 깨짐 없음
□ 단위 표기 정확 (μg/mg/g)
```

#### 2.3 본문 검증 (§2.1 BLOG_RENDER v2.0)
```
□ 글자수 1,620~1,980자
□ H2 4개 이상
□ 키워드 5회 이상
□ 햇살이 언급 2회 이상
□ 경험 공유 톤 사용
□ 급여량 체중별 구분
```

#### 2.4 캡션 검증 (§6.2~6.6)
```
□ 안전 이모지 (🟢/🟡/🔴)
□ CTA 표현
□ AI 고지
□ 해시태그 12~16개
□ 보호자 동질감 메시지
```

#### 2.5 이미지 구성 검증 (§2.1.1)
```
□ 이미지 8장
□ 1장: 표지(한글)
□ 2장: 음식 사진
□ 3~7장: C2 인포그래픽
□ 8장: 햇살이 실사
```

### 3. 결과 출력

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔍 VALIDATOR 결과: {콘텐츠명}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

| # | Validator | 결과 | 상세 |
|---|-----------|------|------|
| 1 | 표지 §8 | ✅ PASS | - |
| 2 | C2 §2.4 | ✅ PASS | 5장 모두 통과 |
| 3 | 본문 §2.1 | ❌ FAIL | 글자수 1,580자 (부족) |
| 4 | 캡션 §6 | ✅ PASS | - |
| 5 | 이미지 구성 | ✅ PASS | 8장 확인 |
| 6 | 8장 실사 | ✅ PASS | 햇살이 실사 확인 |

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
총 결과: {N}/15 PASS

{FAIL 항목 있으면}
❌ FAIL 상세:
- 본문 §2.1: 글자수 1,580자 → 1,620자 이상 필요

→ FAIL 항목 수정 후 재검증 필요
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 스크립트 호출

```bash
# 개별 Validator
python3 .claude/hooks/validators/cover_validator.py <path>
python3 .claude/hooks/validators/c2_validator.py <path>
python3 .claude/hooks/validators/body_validator.py <path>
python3 .claude/hooks/validators/caption_validator.py <path>

# 전체 검증
python3 .claude/hooks/validators/pre_publish_validator.py [folder]
```

## 판정 기준
- **전체 PASS**: 게시 가능
- **1개라도 FAIL**: 게시 불가, 수정 필요
- "괜찮아 보이는데" = 규칙 위반 (§12.1)
