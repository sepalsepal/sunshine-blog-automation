# /body - 본문 슬라이드 생성

## 필수 인자
- `$ARGUMENTS` = 콘텐츠 번호 또는 이름 (예: 060, 후라이드치킨)

## 인자 검증
인자가 없으면 아래 메시지 출력 후 중단:
```
사용법: /body <콘텐츠번호 또는 이름>
예시: /body 060
      /body 후라이드치킨
```

## 실행 절차

### 1. RULES.md 확인
- §2.1 BLOG_RENDER v2.0 확인
- §2.1.1 블로그 이미지 구성 규칙 확인
- §2.4 C2 인포그래픽 v2.0 확인

### 2. 콘텐츠 폴더 찾기
- `contents/` 하위에서 매칭 폴더 검색

### 3. 이미지 구성 (8장)
| 순번 | 이미지 | 유형 |
|------|--------|------|
| 1장 | 표지 | 한글 표지 (§8) |
| 2장 | 음식 사진 | 실사/AI |
| 3~7장 | 인포그래픽 | C2 v2.0 규칙 |
| 8장 | 햇살이 마무리 | **실사 전용** |

### 4. C2 인포그래픽 생성 (3~7장)
각 슬라이드별:
- 영양성분 (SAFE 뱃지, 성분+효능+수치)
- 급여방법 (DO/DON'T, 초록✅/빨강❌)
- 급여량 (표 형태, 주의사항 박스)
- 주의사항 (번호+제목+설명, 응급 안내)
- 조리방법 (STEP 번호, TIP 박스)

### 5. 본문 텍스트 생성
- §6.3 보호자 동질감 규칙 적용
- §6.4 팩트체크 필수 항목 포함
- §6.6 급여량 작성 형식 준수

### 6. Validator 자동 실행
- body_validator.py (글자수, H2, 동질감)
- c2_validator.py (해상도, 레이아웃)

### 7. 결과 보고
```
/body 결과
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
콘텐츠: {번호}_{이름}
생성 이미지: 8장
본문 글자수: {N}자 (1,620~1,980)
H2 개수: {N}개 (4개 이상)
Validator: PASS/FAIL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

| 위치 | 폰트 | 크기 |

| 헤더 부제목 | NotoSansCJK-Regular | 24px |
| 카드 제목 | NotoSansCJK-Bold | 36px |
| 카드 설명 | NotoSansCJK-Regular | 22px (최소) |
| 수치/강조값 | NotoSansCJK-Bold | 44px |
| 하단 주석 | NotoSansCJK-Regular | 18px (최소) |

## 금지 사항
- 8장에 AI 이미지 사용 금지 (실사만)
- 번호+텍스트만 나열 금지
- 하단 보조 요소 없음 금지
