# /blog - 네이버 블로그 HTML 준비

## 필수 인자
- `$ARGUMENTS` = 콘텐츠 번호 또는 이름 (예: 060, 후라이드치킨)

## 인자 검증
인자가 없으면 아래 메시지 출력 후 중단:
```
사용법: /blog <콘텐츠번호 또는 이름>
예시: /blog 060
      /blog 후라이드치킨
```

## 실행 절차

### 1. RULES.md 확인
- §10 레드2 확장 모델 확인
- §2.1 BLOG_RENDER v2.0 확인

### 2. 콘텐츠 로드
- 본문 텍스트 (blog/본문.txt)
- 이미지 8장 (blog/*.png)
- 캡션 (caption*.txt)

### 3. HTML 구조 생성

```html
<!-- 상단 요약 박스 -->
<div class="summary-box">
  {결론 먼저 - 한 줄 요약}
</div>

<!-- 본문 -->
<h2>1. {섹션 제목}</h2>
<p>{본문 내용}</p>
<img src="{Cloudinary URL}" />

<!-- 비교표 (해당 시) -->
<table>
  <tr><th>종류</th><th>특징</th><th>급여 가능</th></tr>
  ...
</table>

<!-- FAQ 5개 -->
<h2>자주 묻는 질문</h2>
<div class="faq">
  <h3>Q. {질문}</h3>
  <p>A. {답변}</p>
</div>

<!-- 간단 레시피 -->
<h2>간단 레시피</h2>
<ol>
  <li>{단계}</li>
</ol>

<!-- 출처 박스 -->
<div class="source-box">
  <p>참고 자료: {출처}</p>
</div>
```

### 4. Cloudinary 이미지 업로드
- 기존 URL 확인 후 신규만 업로드
- HTML에 URL 삽입

### 5. Validator 실행
- body_validator.py (글자수, H2, 동질감)

### 6. 결과 출력
```
/blog 결과
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
콘텐츠: {번호}_{이름}
HTML 파일: blog/blog_preview.html
이미지: {N}장 (Cloudinary URL 적용)
Validator: PASS/FAIL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📋 복사용 HTML:
[HTML 코드 블록 출력]
```

## 레드2 확장 요소 (§10)

| 요소 | 설명 | 필수 |
|------|------|------|
| 상단 요약 박스 | 결론 먼저 | ✅ |
| 비교표 | 종류별/옵션별 | 해당 시 |
| FAQ 5개 | 검색 질의 대응 | ✅ |
| 간단 레시피 | 조리법 1개 | ✅ |
| 출처 박스 | 전문성 표현 | ✅ |

## 블로그 제목 규칙 (§6.5)
- 형식: **"강아지 {음식명} 먹어도 되나요?"**
- 질문형 고정
- 금지: 노령견, 급여량, 주의사항, 완벽 정리 등
