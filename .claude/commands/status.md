# /status - 현재 상태 확인

## 인자
- 인자 없음 (전체 상태 조회)
- 또는 `$ARGUMENTS` = 특정 콘텐츠 번호/이름 (상세 조회)

## 실행 절차

### 1. 전체 상태 조회 (인자 없음)

#### 1.1 contents/ 폴더 스캔
```
contents/
├── 1_cover_only/     ← 표지만
├── 2_body_ready/     ← 본문 완료
├── 3_approved/       ← PD 승인 대기
├── 4_posted/         ← 게시 완료
└── 5_archived/       ← 비활성
```

#### 1.2 각 폴더별 콘텐츠 개수 집계

#### 1.3 결과 출력
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 PROJECT SUNSHINE 현재 상태
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

| 폴더 | 개수 | 상태 |
|------|------|------|
| 1_cover_only | N개 | 표지만 |
| 2_body_ready | N개 | 본문 완료 |
| 3_approved | N개 | PD 승인 대기 |
| 4_posted | N개 | 게시 완료 |

📌 다음 작업 추천:
- {2_body_ready 또는 3_approved에 있는 콘텐츠 안내}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 2. 특정 콘텐츠 상세 조회 (인자 있음)

#### 2.1 콘텐츠 폴더 찾기
- `$ARGUMENTS`로 매칭

#### 2.2 상세 정보 출력
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 콘텐츠 상세: {번호}_{이름}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

위치: contents/{상태폴더}/{콘텐츠폴더}
상태: {상태}

파일 현황:
□ 표지 (01_*.png): {있음/없음}
□ 음식사진 (02_*.png): {있음/없음}
□ 인포그래픽 (03~07_*.png): {N}개
□ 햇살이 (08_*.png): {있음/없음}
□ 본문 (본문.txt): {있음/없음}
□ 캡션 (caption*.txt): {있음/없음}

Validator 상태:
{최근 검증 결과 또는 "검증 필요"}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 상태별 의미

| 폴더 | 의미 | 다음 작업 |
|------|------|----------|
| 1_cover_only | 표지만 완성 | /body 실행 |
| 2_body_ready | 본문 완료 | PD 검토 요청 |
| 3_approved | PD 승인됨 | /publish 실행 |
| 4_posted | 게시 완료 | 완료 |
| 5_archived | 비활성 | - |
